# Session 2026-03-08 — Affichage dans les sections spec, task, output

## Tâche 1: Investigate root cause ✓

### Analyse du problème

En explorant le code `web/static/app.js`, j'ai identifié la racine du problème :

1. **Auto-refresh constant** (ligne 127) : `setInterval(refresh, 3000)` appelle `refresh()` toutes les 3 secondes
2. **Recréation du DOM** : Chaque `refresh()` appelle `renderTabContent()` qui recrée complètement le contenu des onglets
3. **Force-scroll en Output** : La fonction `renderOutputPane()` et les handlers de streaming utilisent `pre.scrollTop = pre.scrollHeight` (7 occurrences) pour forcer le scroll en bas
4. **Perte de position utilisateur** : Quand l'utilisateur remonte manuellement dans Output/Spec/Tasks, le polling suivant (3s max) recrée le DOM et remet le scroll en bas

### Mécanisme identifié

```
[User scrolls up] → 3 sec plus tard → refresh() → renderTabContent()
→ renderOutputPane() → pre.scrollTop = pre.scrollHeight → force scroll down
```

## Tâche 2: Implement fix ✓

### Solution implémentée

**1. Détection du scroll utilisateur** (nouveau code aux lignes 89-114)
```javascript
function setupScrollDetection() {
  ['tab-spec', 'tasks-scroll', 'output-pre'].forEach(id => {
    const el = $id(id);
    if (!el) return;

    el.addEventListener('scroll', () => {
      state._autoRefreshBlocked = true;
      state._userScrolling = id;

      // Resume after 5 seconds of inactivity
      scrollTimeout = setTimeout(() => {
        state._autoRefreshBlocked = false;
        state._userScrolling = null;
      }, 5000);
    }, { passive: true });
  });
}
```

**2. État de blocage du polling** (nouvelles variables dans `state`)
```javascript
state._userScrolling: null,        // id du tab en cours de scroll
state._autoRefreshBlocked: false,  // flag de blocage global
```

**3. Blocage du refresh quand utilisateur scroll** (ligne ~130)
```javascript
async function refresh() {
  // ... sidebar update ...
  if (state._autoRefreshBlocked) return;  // ← SKIP panel re-render
  // ... panel re-render ...
}
```

**4. Auto-scroll sélectif en Output** (fonction helper aux lignes 116-123)
```javascript
function _autoScrollOutput() {
  const pre = $id('output-pre');
  if (!pre) return;
  if (state._userScrolling !== 'output-pre') {
    pre.scrollTop = pre.scrollHeight;
  }
}
```

Cette fonction remplace les 7 appels directs à `pre.scrollTop = pre.scrollHeight`.

## Résultat final

**Le comportement attendu est :**

1. **Spec/Tasks/Output** : Quand l'utilisateur remonte (scroll up), le polling se pause pendant 5 secondes
2. **Output streaming** : Pendant une task run, le contenu se met à jour en temps réel mais ne force pas le scroll si l'utilisateur remonte
3. **Auto-scroll resumption** : Après 5 secondes d'inactivité du scroll, l'auto-refresh reprend normalement
