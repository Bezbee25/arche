arche — documentation

        
      

      


        

          
What is arche?

          
arche keeps your project context alive between sessions. It breaks work into Tracks and Tasks, then runs each task by automatically feeding the full context (spec, architecture memory, history) to your LLM CLI (claude, gemini…).

        


        

          
Core concepts

          

            
TrackA unit of work (feature, bugfix, refactor…). Has its own spec, tasks, and session log. Only one track is ACTIVE at a time.

            
TaskA single step inside a track. arche builds the full context and calls your LLM to execute it.

            
spec.mdGoal + requirements for the track — written once, used as context for every task run.

            
archi.mdArchitecture memory — automatically updated after each task run. Grows richer over time.

          

        


        

          
Task statuses

          

            
· TODONot started yet.

            
▶ IN PROGRESSCurrently being worked on. Only one task at a time. Set automatically when you click ▶ Run.

            
✓ DONECompleted. Use ✓ Done in the action bar or arche task done.

            
✗ BLOCKEDWaiting on something external. Use ✗ Block with a reason.

          

        


        

          
Web UI — action bar

          

            
▶ RunSet task IN PROGRESS and stream LLM output to the Output tab.

            
✎ EditEdit task title, description, and notes.

            
✓ DoneMark the selected task as DONE (with optional notes).

            
✗ BlockMark task as BLOCKED with a reason.

          

        


        

          
CLI — quick reference

          

            
Setup

            
arche initConfigure project + choose LLM model per phase

            
arche webStart the web UI (port 7331)

            
Tracks

            
arche track new "feat: …"Create track — spec Q&A → task generation

            
arche track listList all tracks with progress bars

            
arche track switch <name>Switch active track

            
arche track planGenerate tasks from existing spec

            
arche track doneMark active track as complete

            
Tasks — main loop

            
arche task run★ Run current task with full context → LLM

            
arche task run --auto-doneRun + mark DONE automatically

            
arche task doneMark done, advance to next

            
arche task block "reason"Mark current task blocked

            
arche task add "title"Add a task manually

            
arche task switch <n|id|title>Jump to a specific task

            
arche task listList all tasks

            
Spec

            
arche spec qaQ&A interview → spec + LLM refinement

            
arche spec refineRefine existing spec with LLM (no Q&A)

            
arche spec showDisplay the current spec

            
LLM ad-hoc

            
arche dev "instruction"Free-form coding with track context

            
arche debug "error"Analyse error + fix with track context

            
arche docGenerate documentation (Gemini large context)

            
arche log "note"Add a manual note to the session journal

          

        


        

          
Storage layout

          
storage/tracks/{track-id}/
├── spec.md       ← track goal and requirements
├── archi.md      ← architecture memory (auto-updated)
├── tasks.yaml    ← tasks, statuses, notes
└── sessions/     ← {date}.md — full LLM output per session
